How It Was Built

PHASE 1 — FOUNDATION

🖥️ Ubuntu Server Setup on Consumer Laptop

Installed Ubuntu Server 24 on a repurposed consumer laptop (Intel Celeron N2830, 8.2GB RAM). Configured SSH access, set a static local IP (192.168.1.130), and set up Tailscale for secure remote access from anywhere in the world without exposing SSH to the internet.

Ubuntu Server 24 SSH Static IP Tailscale Netplan
PHASE 2 — CUSTOM DASHBOARD

🏠 HomeBase — Custom GUI Dashboard

Built a full-stack web dashboard from scratch inspired by ZimaOS. The Node.js backend interfaces directly with the Linux kernel via system commands to serve real-time data. The React frontend provides a live, responsive UI accessible from any device on the network or via Tailscale.

Node.js React + Vite Express.js systemd service REST API
PHASE 3 — FEATURES

⚙️ Dashboard Features Built

Iteratively built out all major features: live CPU/RAM/temperature monitoring, drive detection and mount/unmount, full file manager with drag-and-drop, upload/download, thumbnails for images and videos, music player with 575+ songs, network speed test and device scanner, and service management for Plex, Syncthing, and SSH.

File Manager Drive Management Music Streaming Photo Gallery Network Scanner Speed Test
PHASE 4 — MEDIA SERVER

📺 Plex Media Server + Auto-Backup

Deployed Plex Media Server in Docker with custom bind mounts to a 2TB WD external drive containing movies, TV shows, and music. Set up udev rules for automatic drive detection and Plex restart on reconnect. Configured Syncthing for automatic photo backup from Janet's Samsung Galaxy S25+ when on the home network.

Docker Plex Syncthing udev rules exFAT
PHASE 5 — SECURITY HARDENING

🛡️ Full Security Hardening + IDS Dashboard

Hardened the server against real-world attacks: enabled UFW firewall with minimal open ports, closed SSH from the internet entirely (Tailscale-only), generated and deployed SSH key pairs for all devices, configured Fail2Ban for 24-hour IP bans after 3 failed attempts, and built a live security monitoring dashboard with geolocation of attackers and push notifications to phone on ban events.

UFW Firewall Fail2Ban SSH Hardening Key-Based Auth Ntfy Alerts GeoIP Lookup

🔐 Zero Attack Surface Design

SSH is completely invisible to the public internet — only accessible through the Tailscale encrypted mesh network. The firewall blocks all incoming traffic by default. The only public-facing ports are Plex (for media streaming) and Tailscale's UDP port. Fail2Ban watches all authentication logs and automatically bans any IP that triggers suspicious activity, with instant push notifications to the owner's phone.
